## Фаза 4 — History & Stats Agent ✅

> Цель: накапливать статистику по результатам Risk Agent, строить тикер-листы.

- [x] Модуль `agents/history_stats.py`
  - [x] Агрегация P&L по тикерам: сумма, win rate, кол-во сделок
  - [x] Агрегация P&L по дням
  - [x] Фиксация нарушений лимитов (warning/stop/trade_red events)
  - [x] Формирование тикер-листов: зелёный / жёлтый / чёрный / other
    - [x] Зелёный: прибыльные, win_rate ≥ 60%, avg_mpu > −5%
    - [x] Жёлтый: avg_mpu от −5% до −10%, или win_rate < 40%, или missed_tp > 40%
    - [x] Чёрный: убыток при ≥3 сделках, или avg_mpu ≤ −10%
- [x] Хранилище: JSON-файл `data/stats.json` (путь через `settings.stats_data_path`)
- [x] Дедупликация сделок по trade ID — повторный вызов безопасен
- [x] `action='rebuild'` — перестройка из TTM за N торговых дней
- [x] Интеграция в реестр: новый инструмент `history_agent`
- [x] Обновление системного промпта (авто-предложение сохранять риск-отчёты)

## Фаза 5 — Daily Summarizer ✅

> Цель: дневные и еженедельные отчёты с ключевыми уроками.

- [x] Модуль `agents/daily_summarizer.py`
  - [x] Дневной отчёт (`action='daily_report'`):
    - [x] P&L за день vs план (5%), разница
    - [x] Лучшая и худшая сделка дня
    - [x] Нарушения лимитов: STOP, WARNING, over-volume, серия убытков
    - [x] Пропущенные TP (MPP≥5%, закрыт далеко ниже)
    - [x] 1-2 урока на завтра (LLM-генерация через Claude)
  - [x] Недельный отчёт (`action='weekly_report'`):
    - [x] P&L по дням за последние 5 торговых дней (таблица с флагами)
    - [x] Итого: win rate, кол-во сделок, сумма P&L
    - [x] Топ-3 лучших и топ-3 худших тикеров
    - [x] Количество дней с нарушениями лимитов
    - [x] 2-3 урока на следующую неделю (LLM-генерация)
  - [x] Graceful degradation — уроки без ошибок если LLM недоступен
- [x] Интеграция в реестр (`daily_summarizer` tool)
- [x] Обновление системного промпта
- [x] Команды: "Итоги дня", "Дневной отчёт за 4 марта", "Сделай weekly summary"

## Фаза 7 — Веб-дашборд (Trading Command Center) ⬜

> Цель: единый интерфейс трейдера в браузере.

### 7.1 Общая архитектура UI

- [ ] Выбор фреймворка: React (Vite) или статический HTML + Alpine.js
- [ ] Коммуникация с бэкендом: REST + WebSocket (уже есть в оркестраторе)
- [ ] Базовый layout: Header / Sidebar / Main / Footer

### 7.2 Header

- [ ] Статус агента (online / offline / thinking)
- [ ] Текущее время UTC (обновление раз в секунду)
- [ ] Текущий баланс депозита (из TTM или вручную)
- [ ] Счётчик алертов (нарушения лимитов, триггеры Psych Agent)

### 7.3 Chat Interface

- [ ] Чат-окно с историей сообщений
- [ ] Стриминг ответов через WebSocket
- [ ] Отображение инструментов (tool calls) в виде карточек
- [ ] Быстрые кнопки-подсказки: "Утренний чек-ин", "Скан рынка", "Риск-отчёт"

### 7.4 Market Strip

- [ ] Список сделок за сегодня (из TTM): тикер, P&L, время
- [ ] Список кандидатов от сканера: символ, NATR, volume, move_5m
- [ ] Обновление по кнопке или авто-refresh раз в 30 сек

### 7.5 Risk Panel

- [ ] Текущий дневной P&L % (термометр: зелёный / жёлтый / красный)
- [ ] Прогресс к дневной цели (+5%)
- [ ] Индикатор лимитов: -10% (warning) / -15% (stop)
- [ ] Последний риск-отчёт (сворачиваемый блок)

### 7.6 Psych Panel

- [ ] Текущий режим дня (normal / cautious / rest)
- [ ] Последний чек-ин (дата, самочувствие, сон)
- [ ] Кнопки: "Утро", "Вечер", "Тревога" (быстрый ввод в чат)

### 7.7 History Panel

- [ ] Тикер-листы: зелёный / жёлтый / чёрный (таблица)
- [ ] Мини-график P&L по дням за последние 2 недели
- [ ] Кнопка "Недельный отчёт"

## Фаза 8 — Интеграция и полировка ⬜

- [ ] Единый системный промпт учитывает все агенты и их состояния
- [ ] Psych Agent передаёт `trading_mode` флаг в Risk Agent при каждом вызове
- [ ] Daily Summarizer использует History Stats для генерации уроков
- [ ] Все агенты логируют в единый structured log (`logs/office.log`)
